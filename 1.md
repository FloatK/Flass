# WakeUp课程表 Flutter 重写版 — 开发指南 (for AI Agent)

---

## 1. 项目概述

**目标**：实现一款功能完整的课程表 App（参考"WakeUp课程表"的核心体验）。

**技术栈**：

| 类别 | 技术 |
|------|------|
| 框架 | Flutter 3.x + Dart 3 |
| 状态管理 | Riverpod (flutter_riverpod + riverpod_annotation) |
| 本地数据库 | Drift (SQLite) |
| 路由 | go_router |
| 数据序列化 | freezed + json_serializable |
| 网络 | dio (备用) |
| 通知 | flutter_local_notifications |
| 分享 | share_plus |

**架构**：Clean Architecture 分层（core → data → domain → presentation），按功能模块组织。

#### 3.2 数据模型

**你的任务**：在 `lib/data/models/course.dart` 中用 freezed 生成如下模型：

```dart
import 'package:freezed_annotation/freezed_annotation.dart';
import 'package:uuid/uuid.dart';

part 'course.freezed.dart';
part 'course.g.dart';

@freezed
class Course with _$Course {
  const factory Course({
    @Default(Uuid().v4()) String id,
    required String name,
    required String teacher,
    String? location,
    @Default(0xFF2196F3) int color,
    @Default([]) List<TimeDetail> timeDetails,
  }) = _Course;

  factory Course.fromJson(Map<String, dynamic> json) => _$CourseFromJson(json);
}

@freezed
class TimeDetail with _$TimeDetail {
  const factory TimeDetail({
    required int dayOfWeek,         // 1=Mon, 7=Sun
    required int startPeriod,       // 1-12
    @Default(1) int duration,
    @Default([]) List<int> weeks,
    @Default('all') String singleOrDouble, // 'all', 'single', 'double'
  }) = _TimeDetail;

  factory TimeDetail.fromJson(Map<String, dynamic> json) => _$TimeDetailFromJson(json);
}
```

执行 `dart run build_runner build` 生成序列化代码。确保 uuid 包已添加。

#### 3.4 数据库与仓库

**你的任务**：实现 Drift 数据库和仓库。

##### 3.4.1 数据库 (`lib/data/datasources/database.dart`)

- 定义 `Courses` 表：`id(text PK)`, `name(text)`, `teacher(text)`, `location(text nullable)`, `color(int)`
- 定义 `TimeDetails` 表：`id(int PK auto)`, `courseId(text FK → courses.id)`, `dayOfWeek(int)`, `startPeriod(int)`, `duration(int)`, `weeks(text, 逗号分隔)`, `singleOrDouble(text)`
- 创建 DAO：
  - `Stream<List<CourseWithDetails>> watchAllCourses()`
  - `Future<void> insertCourse(...)`
  - `Future<void> updateCourse(...)`
  - `Future<void> deleteCourse(String id)`

> **注意**：因为 weeks 是 `List<int>`，存储时转换为逗号分隔的字符串，读取时还原；timeDetails 需先存储 Course，再存储它的 TimeDetails，并在查询时用 JOIN 组装成 Course 模型。

##### 3.4.2 仓库接口 (`domain/repositories/course_repository.dart`)

```dart
Stream<List<Course>> watchAllCourses();
Future<void> addCourse(Course course);
Future<void> updateCourse(Course course);
Future<void> deleteCourse(String id);
```

##### 3.4.3 仓库实现 (`data/repositories/course_repository_impl.dart`)

实现接口，注入 AppDatabase，完成 Drift 实体 ↔ Course 模型的转换。

##### 3.4.4 验证

运行 `build_runner` 生成数据库代码，解决可能出现的错误。

#### 3.6 周视图页面

**你的任务**：创建 `lib/presentation/pages/week_schedule_page.dart`。

##### UI 布局

- **顶部栏**：显示"2025年春 第X周 (MM.dd - MM.dd)"，左右箭头可切换周次。
- **主体**：一个 8列 × 12行 的网格（首列为节次标签 1-12，剩余7列为周一至周日）。
- **每个课程格子**：
  - 根据课程 `startPeriod` 和 `duration` 占对应行数。
  - 显示课程名和教室，背景色使用课程的 `color`。
  - 空格子显示浅灰色占位。
- **点击课程格子**：底部弹出课程详情（`ModalBottomSheet`），展示教师、教室、周次、节次等。
- **悬浮按钮(FAB)**：跳转到添加课程页面。

##### 数据

从 `courseListProvider` 读取课程，按当前周过滤（需实现周次判断逻辑：根据开学日期计算当前周，或先写死一个基准周）。

#### 3.7 添加/编辑课程页面

**你的任务**：创建 `lib/presentation/pages/add_edit_course_page.dart`。

- 接受可选参数 `courseId`（路由参数），存在则为编辑模式，加载现有数据填充表单。
- **表单字段**：课程名、教师、教室(选填)、颜色选择器(预设几种颜色，可选自定义)。
- **"时间设置"区域**：动态列表，每项包含：
  - 星期选择（下拉）
  - 起始节次（1-12）
  - 持续节数（1-3）
  - 周次选择：1-20周的复选框，并提供"全选"、"单周"、"双周"快捷按钮。
  - 删除按钮移除该时间段。
- 可添加多个时间段（按钮"+ 添加时间段"）。
- 保存时组装 `Course` 对象，调用 `courseListProvider` 的添加/更新方法，返回上一页。
